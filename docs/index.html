How It Works

Four steps between Claude's intent and execution

1

Claude runs a command

Claude Code calls a tool like Bash, Write, or an MCP tool.

2

Hook intercepts

The PreToolUse hook checks against your rules. If matched, it creates an approval request.

3

Browser opens

An approval page opens with the command details and an AI-generated risk description.

4

You decide

Click Approve or Deny. Claude proceeds or stops. Full audit trail preserved.

Features

⚡

Glob Pattern Rules

Define rules with tool name + glob pattern. rm *, git push*, curl * | * — match exactly what you want to gate.

✅

AI Descriptions

Each approval request includes an AI-generated explanation of what the command does and its potential risks.

🔒

JWT-Protected Actions

Approve/deny actions are protected by single-use JWT tokens with 5-minute TTL. No replay, no tampering.

🔌

MCP Tool Gating

Gate any MCP tool with the same rule system. Control third-party tools like databases, APIs, or deployment services.

🚀

Web Dashboard

Real-time dashboard showing all approval requests with status, timestamps, and links to detail pages.

🛠

Fully Extensible

Swap out SQLite for your own API, or replace the entire server with your own approval flow. Your rules.

Simple Configuration

One JSON file controls everything

.hitl.json
{
  "approval": {
    "url": "http://localhost:9457",
    "pollIntervalMs": 1000,
    "timeoutMs": 300000
  },
  "rules": [
    { "tool": "Bash", "pattern": "rm *" },
    { "tool": "Bash", "pattern": "git push*" },
    { "tool": "Bash", "pattern": "curl * | *" },
    { "tool": "mcp__server__tool", "pattern": "*" }  // gate MCP tools too
  ]
}

Three Ways to Run

Use the defaults, customize the storage, or build your own

Default

Local Server + SQLite

Run claude-hitl local and you're done. Approval requests stored in SQLite at ~/.hitl/hitl.db, served through a polished web UI.

Custom Storage

Local Server + Your API

Keep the web UI but swap SQLite for your own backend. Set dataSource.type: "api" in config and point it at your storage API.

Custom Server

Your Own Server

Replace everything. Build your own approval server with your own UI, storage, and flow. The hook just needs two endpoints.
